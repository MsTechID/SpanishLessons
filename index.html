Voz y sonido

Each phrase plays your own recording if one exists; otherwise your device speaks it live.

Recorded clips are identical everywhere and don’t need any voice installed — best for phones and offline. Record each phrase, name the file after its slug, and drop it in an audio/ folder next to this page (or send them to me to embed).

How recording works
  • The CSV lists every phrase, its English, and its exact filename (e.g. como-andas.mp3).
  • Record in Audacity → export each as MP3 (mono is fine) → name it exactly as the CSV says.
  • Put them in a folder called audio beside this HTML file, then host it (your GitHub Pages works great).
  • Or send me the clips and I’ll embed them straight into the file so it stays a single portable page.
  • Recorded audio also gets the slow/normal speed control (pitch kept natural) and volume.
Add or preview a Spanish voice per device
  • iPhone / iPad — preview before using: Settings → Accessibility → Spoken Content → Voices → Spanish → tap Spanish (Argentina) and the ▶ to hear a sample; tap the cloud to download it. That downloaded voice is exactly what this page will use.
  • Fastest check: open this page on the iPhone, pick that voice above, and tap Escuchar la voz — same engine, same sound.
  • Mac: System Settings → Accessibility → Spoken Content → System Voice → Manage Voices → Spanish (Argentina).
  • Android: Settings → Accessibility → Text-to-speech → install Google TTS + a Spanish voice.
  • Windows: Settings → Time & Language → add Spanish → download speech.
  • Linux / GrapheneOS (FOSS): device needs a Spanish speech engine; a Piper/RHVoice speech-dispatcher backend beats default espeak. Recorded clips avoid this entirely.
